# feedback-parser

Claude Code skill для парсинга неструктурированной обратной связи в единый JSON-формат, готовый для разработки.

## Что делает

Превращает сырой фидбэк (заметки с встреч, таблицы, переписки, экспорт из Excel/CSV) в структурированный JSON с классификацией по типу, приоритету, области и сложности. Каждый айтем содержит `dev_notes` — конкретные рекомендации для разработчика.

**Поддерживает:**
- Любой формат входных данных (текст, таблица, CSV, копипаст)
- Смешанные языки (русский, казахский, английский)
- Автоматическое разделение составных пунктов на отдельные айтемы
- Классификацию: `bug`, `ux_issue`, `feature_request`, `question`, `general_feedback`
- Приоритизацию: `p1`–`p4`
- Привязку к областям платформы: `boards`, `tasks`, `documents`, `notifications` и др.

## Использование

В сессии Claude Code вызовите:

```
/feedback-parser
```

Затем вставьте сырые данные с фидбэком. Skill автоматически:

1. Определит формат входных данных
2. Извлечёт каждый отдельный айтем
3. Классифицирует и структурирует в JSON
4. Добавит технические заметки для разработчиков
5. Выведет сводку и рекомендации

## Пример

**Вход** — сырые заметки с встречи:

> Клиент жалуется что доска задач тормозит когда больше 50 карточек. Особенно на мобилке вообще невозможно работать.
>
> Кнопка "Сохранить" на странице настроек иногда не работает. Нажимаешь — ничего не происходит.
>
> При экспорте отчёта в Excel слетает форматирование, колонки съезжают. Критично — отчёты идут руководству.

**Выход** — структурированный JSON:

```json
{
  "meta": {
    "source": "Заметки с встречи",
    "total_items": 3,
    "breakdown": {
      "by_type": { "bug": 3 },
      "by_priority": { "p1": 1, "p2": 2 }
    }
  },
  "items": [
    {
      "id": "FB-001",
      "type": "bug",
      "title": "Тормозит доска задач при 50+ карточках",
      "priority": "p2",
      "area": "boards",
      "dev_notes": "Оптимизировать рендеринг списка карточек (виртуализация). Отдельно проверить мобильную версию."
    }
  ]
}
```

Полный пример: [examples/sample-input.md](examples/sample-input.md) → [examples/sample-output.json](examples/sample-output.json)

## Формат выходных данных

Каждый айтем содержит поля:

| Поле | Описание |
|:-----|:---------|
| `id` | Уникальный ID (`FB-001`, `FB-002`, ...) |
| `type` | `bug` / `ux_issue` / `feature_request` / `question` / `general_feedback` |
| `title` | Краткое описание (5–10 слов) |
| `description` | Подробное описание проблемы или предложения |
| `root_cause` | Причина проблемы (если указана) |
| `area` | Область платформы (`boards`, `tasks`, `documents`, `settings`, ...) |
| `priority` | `p1`–`p4` или `unset` |
| `complexity` | `easy` / `medium` / `hard` / `unset` |
| `dev_notes` | Технические заметки для разработчика |
| `raw_source` | Оригинальный текст для трассируемости |
| `tags` | Теги, включая связи (`duplicate_of:FB-XXX`, `related:FB-XXX`) |
